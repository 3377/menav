## 开发指南

### 框架与语言
> 分析本项目所采用的框架与语言，聚焦最佳实践与规范化要求。

**框架注意事项：**
- 版本兼容性：确保所有依赖与目标框架版本一致
- 特性利用：优先采用框架内置能力，避免重复造轮子
- 性能模式：遵循官方推荐的性能优化模式
- 升级策略：提前规划版本升级路径，降低割裂式改动
- 框架重点备注：
	* {重要说明}

**语言最佳实践：**
- 类型安全：在可行范围内通过类型约束降低运行期错误
- 现代特性：合理使用语言新特性并兼顾兼容性
- 一致性：保持统一的编码范式与结构
- 文档化：记录语言相关的实现细节与权宜之计

### 代码抽象与复用
> 开发过程中优先考虑抽象与复用，保障功能模块化、组件化。动手前先检索既有方案，避免重复实现。
> 在下方列出项目中常用组件、工具函数、API 封装等目录结构。


**模块化设计原则：**
- 单一职责：每个模块仅负责一个功能点
- 高内聚低耦合：相近能力集中管理，减少跨模块依赖
- 稳定接口：对外暴露稳定接口，内部实现可迭代

**可复用组件库示例：**
```
root
- pkg 
    - utils
		- time // 包含时间工具：getNow、getToday、getYesterday 等
    - views
```
