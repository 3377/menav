# 多层级书签配置说明

## 功能实现总结

### 1. 书签页面层级优化

**配置位置：** `config/user/site.yml`

```yaml
bookmarks:
  # 跳过顶层空分类，直接显示下一层
  skipEmptyTopCategory: true
  
  # 默认展开的层级深度
  # 0 = 全部收起
  # 1 = 展开第一层（收藏夹栏展开，显示下面的groups）
  # 2 = 展开到第二层（groups也展开，显示里面的书签）
  defaultExpandDepth: 1
```

**效果：**
- ✅ 第一级标题显示为"收藏夹栏"（跳过了空字符串分类）
- ✅ "收藏夹栏"默认展开，显示下面的目录（个人、工具类等）
- ✅ 这些目录（groups）默认收起，不展开里面的书签

### 2. 首页动态书签展开

**配置位置：** `config/user/site.yml`

```yaml
homepage:
  bookmarkFolder: ["", "收藏夹栏", "常用导航"]
  enabled: true
  # 首页分类默认展开
  defaultExpanded: true
```

**效果：**
- ✅ 首页显示"常用导航"文件夹的内容
- ✅ "常用导航"分类默认展开，显示里面的书签

## 配置说明

### skipEmptyTopCategory（跳过空顶层分类）

- `true`：跳过第一层空名称的分类，直接显示subcategories
- `false`：保持原有的多层级结构

**适用场景：**
浏览器书签导出时，第一层通常是空名称的根目录，跳过后可以让界面更简洁。

### defaultExpandDepth（默认展开层级深度）

从显示的第一层开始计算：

- `0`：全部收起
- `1`：展开第一层（收藏夹栏）
- `2`：展开到第二层（groups）
- `3+`：继续展开更深层级

**注意：**
- 用户手动切换的状态会保存在localStorage中，优先级更高
- 清除浏览器缓存后，会重新应用配置的默认值

### homepage.defaultExpanded（首页默认展开）

- `true`：首页的分类默认展开
- `false`：首页的分类默认收起

## 自定义配置示例

### 示例1：全部展开
```yaml
bookmarks:
  skipEmptyTopCategory: true
  defaultExpandDepth: 3  # 展开所有层级
```

### 示例2：全部收起
```yaml
bookmarks:
  skipEmptyTopCategory: true
  defaultExpandDepth: 0  # 全部收起
```

### 示例3：不跳过顶层
```yaml
bookmarks:
  skipEmptyTopCategory: false  # 保留原始结构
  defaultExpandDepth: 1
```

### 示例4：首页默认收起
```yaml
homepage:
  bookmarkFolder: ["", "收藏夹栏", "常用导航"]
  enabled: true
  defaultExpanded: false  # 首页分类收起
```

## 技术实现

### 后端（generator.js）
- `skipEmptyTopCategory()` 函数：提取subcategories并重新调整层级
- `extractBookmarkFolder()` 函数：从书签中提取指定路径的内容
- 配置传递到前端：通过 `configJSON` 传递配置

### 前端（script.js）
- `restoreToggleState()` 函数：根据配置和localStorage恢复展开状态
- `initializeNestedCategories()` 函数：初始化所有可折叠元素
- 层级深度判断：根据 `data-level` 属性和配置决定是否展开

## 故障排除

### 配置未生效
1. 确认 `site.yml` 配置正确
2. 重新运行 `npm run build`
3. 清除浏览器缓存和localStorage
4. 刷新页面

### 层级显示错误
1. 检查 `bookmarks.yml` 文件结构
2. 确认第一层是否为空名称分类
3. 查看控制台是否有错误信息

### 展开状态异常
1. 清除localStorage：`localStorage.clear()`
2. 刷新页面重新应用配置
3. 检查 `data-level` 属性是否正确
